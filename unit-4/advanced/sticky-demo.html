Intro Layout Overflow Bug Summary

Intro

Sticky positioning lets an element behave normally until it reaches a defined offset. Then it sticks in place while the surrounding content continues to scroll.

This page is intentionally long so the sticky behavior is easy to test. The table of contents should remain visible as the reader moves down the page.

Layout

The page uses a two-column layout. The left column contains the sticky navigation, and the right column contains the scrolling article sections.

The sticky element includes position: sticky and top: 1rem. Without the top offset, sticky positioning would not visibly activate.

Overflow Bug

Sticky positioning can break if an ancestor element has overflow: hidden, overflow: auto, or overflow: scroll. That ancestor can become the sticky element's scroll container.

The CSS for this page includes a comment showing where that problem was tested and how it was fixed.

Summary

Sticky modules are useful for table of contents menus, section labels, sidebars, comparison tools, and persistent page actions.

This demo completes the sticky positioning requirement for the Unit 4 advanced layout assignment.

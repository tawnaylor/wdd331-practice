This box uses position: sticky and top: 1rem.

Section 1

Overview

Sticky positioning is useful when a page has a helpful module that should stay available while the user scrolls. In this demo, the sticky module acts like a trail guide or table of contents.

The content column is intentionally long so the sticky behavior is easy to see. Scroll down the page and watch the guide remain near the top of the viewport.

Section 2

Packing

A sticky sidebar can help readers keep track of where they are in a long guide. It works especially well for tutorials, product pages, documentation, and long articles.

The sticky element still belongs to the normal document flow. It only sticks after it reaches the offset value defined in CSS.

Section 3

Weather

The required offset is important. If the sticky element does not have a property such as top, bottom, left, or right, then sticky positioning will not visibly activate.

In this demo, the offset is top: 1rem, which keeps the trail guide one rem below the top edge of the viewport.

Section 4

Safety

Sticky positioning can break when an ancestor element has overflow set to hidden, auto, or scroll. That overflow ancestor may become the sticky element's scroll container.

The CSS includes a required comment explaining how the overflow problem was tested and fixed.

Section 5

Wrap Up

This demo shows a practical sticky module that could be used on a real content page. The module stays available while the reader moves through the article.

It also demonstrates the most common sticky troubleshooting issue: overflow applied to the wrong ancestor.
